# Simoola Admin Dashboard 🚀

O **Simoola** é um painel administrativo de alta performance e design premium desenvolvido para a gestão de treinamentos e simulações com Agentes IA. O foco do projeto é oferecer uma experiência fluida (UX) e visualmente impactante (UI) para administradores e analistas.

## ✨ Destaques do Projeto

- **Design Premium:** Estética baseada em *Glassmorphism*, paleta de cores harmoniosa (Simoola Purple) e micro-animações.
- **Dashboard Analítico:** Gráficos interativos (Recharts), KPIs operacionais e widgets de calendário.
- **Central de Simulações:** Gestão unificada de cenários e biblioteca de personas IA através de um sistema de abas intuitivo.
- **Hub CAJ:** Central de busca rápida e atendimento focada na produtividade.
- **Otimização Next.js:** Uso de Metadata API para SEO, Loading States nativos para transições suaves e Server Components para performance.

## 🛠️ Tecnologias Utilizadas

- **Core:** [Next.js 14+](https://nextjs.org/) (App Router)
- **Estilização:** [Tailwind CSS v4](https://tailwindcss.com/)
- **Linguagem:** [TypeScript](https://www.typescriptlang.org/)
- **Ícones:** [Lucide React](https://lucide.dev/)
- **Gráficos:** [Recharts](https://recharts.org/)
- **Datas:** [date-fns](https://date-fns.org/)

## 🚀 Como Instalar e Rodar

### Pré-requisitos
- [Node.js](https://nodejs.org/) (Versão 18 ou superior)
- [Git](https://git-scm.com/)

### Instalação Rápida (Windows)
Se você estiver no Windows, basta executar o arquivo facilitador na raiz do projeto:
```bash
./setup.bat
```
Este script irá instalar as dependências e iniciar o servidor automaticamente.

### Instalação Manual
1. **Clone o repositório:**
   ```bash
   git clone https://github.com/Shamira-Chu/simoola-challenge.git
   cd simoola-challenge
   ```

2. **Instale as dependências:**
   ```bash
   npm install
   ```

3. **Inicie o servidor de desenvolvimento:**
   ```bash
   npm run dev
   ```

4. **Acesse no navegador:**
   Abra [http://localhost:3000](http://localhost:3000)

## 📁 Estrutura de Pastas

- `src/app`: Rotas e páginas da aplicação (Next.js App Router).
- `src/components`: Componentes reutilizáveis organizados por contexto (dashboard, relatorios, etc).
- `public`: Ativos estáticos como favicons e imagens.
